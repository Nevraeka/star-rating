Events

ratingUpdated

rating updated is a DOM Custom Event that fires whenever the value of the <star-rating> is updated. This event passes a JavaScript object with 2 key/value pairs to 'event.detail'.

Event Detail data

maxValue - this is the current maxvalue attribute value at the time the event fires

value - this is the current value property/attribute value at the time the event fires

Attributes

value (RO)

required: n/a (read-only) | expected output: integer | default value: 0

value is mapped directly to the value property & is meant to be a 'output only' attribute. This means it is internally managed by the element and should be considered 'read-only'

img-paths

required: no | expected value: 2 comma separated url strings | default value: (datauri svg embed)

This attribute is 2 urls that are comma separated in a single string value. These 2 img paths get added as background-image url's in the css

maxvalue

required: no | expected value: integer | default value: "5"

maxvalue is the total number of stars you wish to display.

hover

required: no | expected value: boolean | default value: none

When assigned to a value of 'true' this will add a hover handler to th element

size

required: no | expected value: css unit string | default value: "36px"

size defines the height & width values in css recognized units for each individual star

Properties

value

value type: number

Getting the star-rating value

use the 'ratingUpdated' event detail (see the events section) OR use the DOM (document.querySelector('star-rating').value)

the value property represents the number of stars currently selected. Value directly maps to the value attribute (see above).

Methods

reset

this method resets the value of the <star-rating> element to 0 and triggers the ratingUpdated event.
